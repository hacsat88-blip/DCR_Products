# Testing Standards — テスト基準

> origin: ECC common/testing.md (DCR 向けに再構成)
> deploy.ps1: `_` プレフィクスのため deploy 対象外（参照ドキュメント）

## カバレッジ目標

| レベル | 基準 |
|--------|------|
| 最低ライン | 80% カバレッジ |
| 新規コード | 必ずテスト付き |
| バグ修正 | 再現テスト → 修正 → テストパス の順序 |

## テスト種別

| 種別 | 対象 | 速度目標 |
|------|------|----------|
| Unit | 関数・クラス単体 | < 100ms/テスト |
| Integration | モジュール間連携 | < 1s/テスト |
| E2E | ユーザーフロー全体 | < 30s/テスト |

## TDD 原則（tdd-workflow スキルと連携）

```
RED    → 失敗するテストを書く
GREEN  → テストを通す最小の実装
IMPROVE → リファクタ（テスト維持）
VERIFY → 全テスト再実行
```

## テスト品質基準

- **独立性**: テスト間に依存関係を作らない
- **再現性**: 環境や実行順序に依存しない
- **明確性**: テスト名で「何が / どうなるべきか」がわかる
- **速度**: CI で全テスト 5 分以内を目標

## アンチパターン

| やりがち | 正しいアプローチ |
|----------|------------------|
| 実装後にテストを書く | TDD: テストが先 |
| モック過多 | 統合テストで実際の動作を検証 |
| ハッピーパスのみ | エッジケース・エラーケースも必須 |
| スナップショット依存 | 意味のあるアサーションを書く |
