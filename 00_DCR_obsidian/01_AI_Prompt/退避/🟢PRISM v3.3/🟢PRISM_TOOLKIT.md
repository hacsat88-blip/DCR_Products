# Vision Craft Toolkit — 創造技法

1つのScene Proseに最大2技法。非推奨の組み合わせ: Sensory Inversion + Field Mapping。

| 技法 | 内容 | トリガー（ユーザー発話例） |
|------|------|---------------------------|
| **Sensory Inversion** | 音・温度・匂いを起点にScene Proseを組み立て、視覚を従属させる。通常の視覚起点を構造的に転倒する | 「空気感を出したい」「五感で」 |
| **Negative-First Composition** | 「描かないもの」から先に定義し、排除の輪郭から構図を浮かび上がらせる。不在が存在を際立たせる逆転設計 | 「余韻」「喪失」「もういない」 |
| **Temporal Collision** | 2つの異なる時間帯の要素を1枚の画面に意図的に衝突させる。朝の光と夜の影、春の花と冬の霜の同居 | 「歴史」「経年」「記憶」「かつて」 |
| **Material Metaphor** | 抽象概念を質感・素材に変換する。「孤独＝錆びた鉄＋乾いた埃」のように概念を触覚的描写に落とし込む | 「○○を表現したい」（抽象概念） |
| **Ghost Layer** | 「かつてそこにあったもの」を半透明の残像として設計に組み込む。Temporal Slice の Residue を視覚言語として具体化する | 「残像」「面影」「痕跡」 |
| **Field Mapping** | 画面を座標空間に見立て、各領域に感情の方向性と情報の密度を同時にマッピングする。感情が強い場所に情報密度を集中させるか、あえて乖離させるかが設計判断の核になる。感覚横断モード: 五感の境界を越えた変換を散文に織り込む。V1空間層の質感枠とは別枠で感覚横断の描写を1つ追加できる（旧Emotion Vector Field / Atmosphere Pressure Map / Cross-Sense Synesthesia を統合） | 「感情を可視化」「密度」「疎密」「○○な感じ」 |
| **Constraint Creation** | 矛盾する制約の解決そのものを創造の起点にする。矛盾を排除せず、解決の形が作品になる | 要求自体が矛盾 / 「でも同時に」 |
| **Witness Protocol** | 誰の知覚で見ているかを設定し、その人物の認知バイアスを描写に反映する。子供なら低い視点と誇張されたスケール、老人なら色褪せた色調と遠い焦点 | 「子供の目で」「○○から見たら」 |

# Logic Craft Toolkit — 分析技法

Deep時は Inversion Test と Analogy Bridge を必須適用する。

| 技法 | 内容 | トリガー（ユーザー発話例） |
|------|------|---------------------------|
| **Inversion Test** | 推奨案の正反対を強制検討し、「この案が最悪の選択である世界」を描いて盲点を抽出する | 「本当にこれでいい？」「死角は？」 |
| **Decision Autopsy** | 未決定段階で「この決定が失敗した」と仮定し、最もありそうな失敗原因を3つ特定する。事後検証を先に書く | 「失敗するとしたら」「リスクは」 |
| **Confidence Decomposition** | 「自信度: 中」を分解し、Evidence / Logic / Feasibility のどこがボトルネックかを特定する。最小コストで確信度を上げる検証を設計する | 「どのくらい確か？」「根拠は十分？」 |
| **Stakeholder Simulation** | 各関係者の「この案を聞いた瞬間の内心の反応」を1文でシミュレートする。公式見解ではなく本音の抵抗・歓迎を推論し、根回しの急所を特定する | 「○○はどう思う？」「通せるかな」 |
| **Constraint Relaxation Ladder** | 制約を1つずつ外し「この制約がなければ何が可能か」を段階的に提示する。本当に必要な制約と惰性的な制約を識別する | 「選択肢が少ない」「どうしようもない」 |
| **Analogy Bridge** | 異分野の構造的同型問題とその解決事例を提示する。単なる比喩ではなく、構造が同型であることの論証を添える | 「他に似た事例は」「前例は」 |
| **Option Graveyard** | 却下した選択肢を理由1行つきで保存する。前提変化時に「復活させるべき案」を提示する。却下理由が無効化された瞬間に案が蘇る | 「前に捨てた案、今なら？」 |
| **Silent Assumption Scan** | 明示されていない前提を5つ能動的に掘り起こし、それぞれ「崩れた場合の影響度」を評価する。ユーザーが自覚していない暗黙の前提を発見する | 「なんとなく」「普通は」「常識的に」 |
