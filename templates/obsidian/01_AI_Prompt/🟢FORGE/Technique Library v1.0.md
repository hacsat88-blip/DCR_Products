## Vision 創造技法

1つのScene Proseに最大2技法。自明なら黙って適用し末尾に技法名を報告。微妙なら適用せず末尾に1つだけ提案。

| 技法 | トリガー | 効果 |
|------|----------|------|
| Sensory Inversion | 「空気感」「五感」 | 支配的でない感覚（触覚・嗅覚・聴覚）から場面を開き、視覚情報を後から重ねる |
| Negative-First | 「余韻」「喪失」「もういない」 | 不在・欠落から描き始め、かつて在ったものを間接的に浮かび上がらせる |
| Temporal Collision | 「歴史」「経年」「記憶」 | 異なる時間層を同一空間に同居させ、時の厚みを可視化する |
| Material Metaphor | 「○○を表現したい」（抽象） | 抽象概念を具体的な素材・質感・物理現象に置換して描写する |
| Ghost Layer | 「残像」「面影」「痕跡」 | 現在の場面に過去の痕跡を半透明に重ね、二重露光的な奥行を生む |
| Field Mapping | 「感情を可視化」「密度」「疎密」 | 感情や関係性を空間の密度・温度・光量の勾配として物理的に配置する |
| Constraint Creation | 要求が矛盾 / 「でも同時に」 | 矛盾する要求を制約として受け入れ、その緊張から新しい表現を導出する |
| Witness Protocol | 「子供の目で」「○○から見たら」 | 特定の観察者の知覚フィルタを通して場面を再構成する |

### 非推奨の組み合わせ

- **Sensory Inversion + Field Mapping**: 両方が感覚チャンネルを操作するため、描写の焦点が分散する

### 技法の重ね方

- 1技法で十分な場合は1技法に留める（過剰適用は描写を濁らせる）
- 2技法を重ねる場合、一方を構造（場面の骨格）、他方を質感（表面の手触り）に割り当てると干渉しにくい

## Logic 分析技法

Deep時はInversion Test＋Analogy Bridgeを必須実行。他はトリガーに応じて動的適用。

| 技法 | トリガー | 効果 |
|------|----------|------|
| Inversion Test | 「本当にこれでいい？」「死角は？」 | 推奨案が最悪の選択である世界を描き、隠れた前提とリスクを炙り出す |
| Decision Autopsy | 「失敗するとしたら」「リスクは」 | 未来の失敗地点から逆算し、致命的な分岐点を特定する |
| Confidence Decomposition | 「どのくらい確か？」「根拠は十分？」 | 確信度を構成要素（データ量・情報源の質・推論の長さ）に分解し、最も弱い環を特定する |
| Stakeholder Simulation | 「○○はどう思う？」「通せるかな」 | 主要な利害関係者の視点で推奨案を評価し、抵抗点と協力点を予測する |
| Constraint Relaxation | 「選択肢が少ない」「どうしようもない」 | 暗黙の制約を1つずつ仮に外し、解空間を再探索する |
| Analogy Bridge | 「他に似た事例は」「前例は」 | 異分野の構造同型事例を持ち込み、新しい選択肢や評価軸を導入する |
| Option Graveyard | 「前に捨てた案、今なら？」 | 過去に却下した選択肢を現在の条件で再評価する |
| Silent Assumption Scan | 「なんとなく」「普通は」「常識的に」 | 暗黙の前提を掘り起こし可視化する。キーストーン前提の識別はFORGE §6 Mutation Thresholdに委譲 |

### 技法の連鎖

- Silent Assumption Scan → Inversion Test: 掘り起こした前提の中で最も危険なものを反転させる
- Stakeholder Simulation → Constraint Relaxation: 抵抗点が制約になっている場合、その制約を仮に外す
- Decision Autopsy → Analogy Bridge: 特定した失敗パターンに対して、他分野の対処事例を探す

## Table連鎖推奨パターン

Thinking Tableで2型を段階的に重ねる場合の推奨組み合わせ。FORGE §4の「2型目はユーザー承認後」規律に従う。

### 推奨連鎖

| 1型目 | 2型目 | 用途 |
|-------|-------|------|
| Assumption | Risk | 前提が崩れた場合のリスクを評価する |
| Assumption | Tradeoff | 前提整理後に選択肢を比較する |
| Risk | Dependency | リスク緩和策の依存関係を整理する |
| Comparison | Stakeholder | 比較結果に対する利害関係者の反応を評価する |
| Tradeoff | Dependency | 採用案の実行順序を整理する |
| Opportunity | Risk | 機会獲得に伴うリスクを評価する |

### 連鎖の規律

- 1型目の結論を固定点として2型目に引き継ぐ
- 2型目が1型目の結論と矛盾する場合、矛盾を明示しユーザーに判断を仰ぐ
- 2型の重ねで十分な結論に至らない場合、Decision Formatで中間結論を出してから次の分析サイクルを開始する
