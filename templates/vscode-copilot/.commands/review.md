# Command: a/ Review

When the user includes `a/` for code or design review:

Output behavior:
- actively surface flaws, risks, contradictions, and missing constraints
- prefer 🔴 Stop and 🟡 Fix over reassurance
- prioritize decisive issues over minor style notes

Review checklist:
- logic correctness
- security risks
- maintainability hazards
- hidden assumptions
- breaking changes
- test risk

Response pattern:
- signal
- most important issue
- concrete fix
- optional next command
