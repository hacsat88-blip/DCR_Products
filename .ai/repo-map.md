before proposing changes.

# Entrypoints

Each editor/CLI reads its own entry point from the workspace root.
All entry points share the same DCR Kernel philosophy but are formatted
for each tool's reading conventions.

| Editor/CLI          | Entry Point                        | Reads                                    |
|---------------------|------------------------------------|------------------------------------------|
| VS Code Copilot     | .github/copilot-instructions.md    | .ai/, .commands/, rules/, skills/        |
| Copilot CLI         | AGENTS.md → COPILOT_CLI.md         | .ai/repo-map.md, rules/, skills/         |
| Codex               | AGENTS.md                          | rules/ (by reference)                    |
| Claude Code         | CLAUDE.md                          | rules/ (by reference)                    |
| Cursor              | .cursor/rules/dcr-kernel.md + *.mdc| Inlined (generated by deploy.ps1)        |
| Gemini/Antigravity  | .gemini/settings.json              | Inlined systemInstruction                |

# Main Directories

## Execution layer (auto-loaded by each editor)

.github/copilot-instructions.md
VS Code Copilot entry point. DCR Kernel with Permission model (P1/P2/P3),
routing priority, and COPILOT_CLI.md reference.

AGENTS.md
Codex / Copilot CLI entry point. DCR Kernel + module behaviors.
Copilot CLI sessions redirect to COPILOT_CLI.md for full rule set.

COPILOT_CLI.md
GitHub Copilot CLI 専用の完全ルール定義。AGENTS.md から参照される。
Session initialization, tool hierarchy, permission model (P1/P2/P3),
SQL task tracking, error handling, completion verification を含む。

CLAUDE.md
Claude Code entry point. DCR Kernel + safety boundaries + permission model.

.ai/
AI system kernel, repo map, routing design, and behavior modules.

.ai/kernel.md
共通カーネル正本。全エディタの入口ファイルはこの思想を継承。

.ai/repo-map.md
このファイル。プロジェクト構造と規則の参照用。

.ai/rule-routing-design.md
ルール動的参照の設計文書。単一ソース原則、auto-load 安全条件、
エディタ別マッピングを定義。

.ai/module/
Specialized behavior modules (architecture, debugging, review, prompting).

.commands/
Trigger-activated command definitions (a/, i/, r/, s/, d/).

.cursor/rules/dcr-kernel.md
Cursor entry point. DCR Kernel + all module behaviors inlined.

.cursor/rules/*.mdc
Cursor agent rules. deploy.ps1 が rules/*.md から自動生成。

.gemini/settings.json
Gemini/Antigravity entry point. DCR Kernel via systemInstruction.

.vscode/
VS Code workspace settings (useInstructionFiles: true).

## Asset layer (not auto-loaded — referenced on demand)

rules/
Agent-specific behavior rules (84 agents). 正本。
deploy.ps1 で Cursor (.mdc) / VS Code (skills) へ配布。

skills/
Skill definitions (51 skills). Git-managed source of truth.
deploy.ps1 で ~/.agents/skills/ (VS Code Copilot) へ配布。

templates/
Model-specific configuration templates (archive/source).
Subdirs: obsidian/, claude-code/, vscode-copilot/

prototypes/
Experimental projects and proof-of-concepts.

## External references (.gitignore 対象)

copilot-cli-for-beginners/
GitHub Copilot CLI 学習用リファレンスリポジトリ (github/copilot-cli-for-beginners)。
独自の .git/ を持つ外部クローン。DCR 設定とは無関係。

## Deploy

deploy.ps1
一方向同期スクリプト。サトシ開発 → 各エディタのユーザーレベルパスへ配布。
対象: VS Code Copilot (skills), Cursor (rules → .mdc 変換)。
オプション: -Target (all/vscode/cursor), -DryRun, -Check (drift検出)。

deploy.ps1
One-way sync script: skills/ → ~/.agents/skills/ (VS Code Copilot),
rules/*.md (excluding `_*.md`) → generated ~/.cursor/rules/*.mdc + dcr-kernel.md (Cursor).

# Architecture Notes

Preferred layering:
copilot-instructions.md → kernel.md → module/*.md + .commands/*.md

Configuration pattern:
Markdown-based instruction files, no code dependencies.

Deployment pattern:
deploy.ps1 syncs skills/ and rules/ to editor-specific user paths.
One-way only: repository → editor. Never reverse.

---

# Change Rules

When modifying this repository:

- edit skills/ and rules/ here, then deploy
- prefer minimal diffs
- preserve existing naming conventions
- do not create files in templates/ unless archiving a new editor config
- do not modify ~/.agents/ or ~/.cursor/ directly
